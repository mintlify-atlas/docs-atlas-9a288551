Building your own Flipper Application Package (FAP) lets you run custom code directly on your Flipper Zero. FAPs are written in C (or Rust) and compiled against the firmware's API, then loaded from the SD card at runtime.

## Build tools

<CardGroup cols={2}>
  <Card title="uFBT — micro Flipper Build Tool" icon="wrench" href="https://github.com/flipperdevices/flipperzero-ufbt">
    The primary tool for building FAPs. uFBT handles downloading the correct SDK for your target firmware, compiling your app, and deploying it to a connected Flipper.
  </Card>
  <Card title="Arduino IDE" icon="microchip" href="https://www.arduino.cc/en/software">
    Used for companion apps running on ESP32 or ESP8266 modules connected to the Flipper's GPIO header. Not used for FAPs themselves, but common for hardware companion projects.
  </Card>
</CardGroup>

## Setting up your build environment

<Steps>
  <Step title="Install uFBT">
    Follow the installation instructions in the [uFBT repository](https://github.com/flipperdevices/flipperzero-ufbt). uFBT is the officially supported build tool and handles SDK management automatically.
  </Step>
  <Step title="Create your app directory">
    Create a new folder for your project. At minimum you need an `application.fam` manifest file and at least one `.c` source file.
  </Step>
  <Step title="Write your application">
    Implement your app using the Flipper SDK APIs. See the hello world examples below for a starting point, and the official documentation for the full API surface.
  </Step>
  <Step title="Build with uFBT">
    Run `ufbt` in your project directory. uFBT will download the appropriate SDK if needed and produce a `.fap` file in the `dist/` folder.
  </Step>
  <Step title="Deploy to Flipper">
    Copy the resulting `.fap` to your SD card's `apps/` folder, or use `ufbt launch` to build and deploy over USB in one step.
  </Step>
</Steps>

## Basic app structure

Every FAP needs two things: a manifest and at least one C source file.

```c hello_world.c
#include <furi.h>
#include <gui/gui.h>
#include <gui/view_port.h>
#include <input/input.h>

static void draw_callback(Canvas* canvas, void* ctx) {
    UNUSED(ctx);
    canvas_clear(canvas);
    canvas_set_font(canvas, FontPrimary);
    canvas_draw_str(canvas, 10, 32, "Hello, Flipper!");
}

static void input_callback(InputEvent* event, void* ctx) {
    FuriMessageQueue* queue = ctx;
    furi_message_queue_put(queue, event, FuriWaitForever);
}

int32_t hello_world_app(void* p) {
    UNUSED(p);
    FuriMessageQueue* queue = furi_message_queue_alloc(8, sizeof(InputEvent));

    ViewPort* vp = view_port_alloc();
    view_port_draw_callback_set(vp, draw_callback, NULL);
    view_port_input_callback_set(vp, input_callback, queue);

    Gui* gui = furi_record_open(RECORD_GUI);
    gui_add_view_port(gui, vp, GuiLayerFullscreen);

    InputEvent event;
    while(furi_message_queue_get(queue, &event, FuriWaitForever) == FuriStatusOk) {
        if(event.type == InputTypeShort && event.key == InputKeyBack) break;
    }

    gui_remove_view_port(gui, vp);
    furi_record_close(RECORD_GUI);
    view_port_free(vp);
    furi_message_queue_free(queue);
    return 0;
}
```

```python application.fam
App(
    appid="hello_world",
    name="Hello World",
    apptype=FlipperAppType.EXTERNAL,
    entry_point="hello_world_app",
    requires=["gui"],
    stack_size=1 * 1024,
    fap_category="Examples",
)
```

<Note>
  The `entry_point` in `application.fam` must match the function name in your C file. The `appid` must be unique across all installed FAPs.
</Note>
